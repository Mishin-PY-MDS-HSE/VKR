## 3. Моделирование
### 3.1. Формирование наборов признаков
Набор A — базовые характеристики объекта:\
площадь, этаж, количество комнат, год постройки, тип дома, материал стен, наличие балкона и т.д.\
Набор B — минимальная геоинформация:\
район/округ, расстояние до центра, расстояние до ближайшей станции метро\
Набор C — детальные POI-признаки:\
Транспортная доступность (количество остановок в радиусе, время до ключевых узлов)\
Образование (школы, детские сады, вузы)\
Торговля и сервисы (магазины, ТЦ, рестораны, аптеки)\
Рекреация (парки, скверы, фитнес-центры)\
Негативные факторы (промзоны, кладбища, шумные магистрали)

### 3.2. Схема экспериментов
1 A + B (Baseline: объект + базовая локация)\
2 A + B + C (Полная модель с POI)\
3 A + B + C (POI после feature selection)

Каждый эксперимент проводится на нескольких алгоритмах:

Линейная регрессия (Ridge/Lasso)\
EBM (Explainable Boosting Machine)\
CatBoost/LightGBM

Итого 9 комбинаций (3 набора × 3 алгоритма).

### 3.3. Метрики и статистическая значимость
Основные метрики: MAE, MAPE, R²\
Проверка значимости различий:

Кросс-валидация\
Сравнение распределений ошибок\
Доверительные интервалы для метрик


### 3.4. Анализ вклада POI-признаков
Если POI улучшают модель:\
\
SHAP-важность по категориям POI\
Какие объекты окружения влияют сильнее всего?\
На каком радиусе буферной зоны эффект максимален?
\
Если POI не улучшают или ухудшают:
\
Анализ мультиколлинеарности с базовыми признаками\
Проверка гипотезы: район уже «поглощает» информацию о POI\
Возможно, эффект есть только для отдельных сегментов (новостройки vs вторичка, центр vs окраины)
